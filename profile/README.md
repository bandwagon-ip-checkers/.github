## 在开始之前：先搞清楚"被封"的两种情况

很多人以为 IP 被封是一件事，但其实它分两种，处理方式完全不同。

**第一种：被搬瓦工官方封禁**。这种一般是因为严重违反了服务条款，比如长时间高负载跑垃圾邮件、挖矿、滥用资源之类的。后果是 KiwiVM 面板里直接显示 IP BLOCKED，通常只能付费更换 IP。

**第二种：被国内网络防火墙屏蔽（俗称"被墙"）**。这种情况下 VPS 本身运转正常，境外访问没问题，只是从国内连不进去。这是最常见的情况，解决方式是换 IP 或换机房。

判断清楚是哪种情况，是后续处理的前提。

这是最权威的方法，直接告诉你 IP 有没有被搬瓦工官方列入黑名单。

**操作步骤如下：**

1. 登录搬瓦工账号，进入 Services → My Services，找到你需要检测的那台 VPS，点击 KiwiVM Control Panel 按钮。

2. 进入 KiwiVM 面板后，在浏览器地址栏找到当前 URL，手动将末尾的 `main.php` 修改为 `main-exec.php?mode=blacklistcheck`，回车加载页面。

**检测结果解读：**

- **IP NOT BLOCKED**：IP 在搬瓦工官方层面完全正常，没有被列入黑名单。如果还是连不上，可能是"被墙"或者 VPS 本身没开机。

## 方法二：第三方多节点连通性检测（判断是否"被墙"）

这是判断"IP 是否被墙"最直观的方法。原理很简单：如果境外节点能访问，但国内节点全部不通，那基本可以确认 IP 被墙了。

**推荐使用的工具：**

**ping.pe** — 输入你的 VPS IP 地址，会自动从全球多个节点（包括中国大陆的多个节点）发起 ping 测试。结果用绿色（通）和红色（不通）显示，一目了然。境外全绿、国内全红，那就是被墙了。

**IP112.cn** — 支持同时测试 ICMP（ping）和 TCP 端口连通性。输入 IP 和端口号（比如 SSH 端口），可以分别看到国内和国外的连通情况。"TCP 可用"表示 TCP 连接正常，"ICMP 可用"表示可以 ping 通。

**使用步骤：**
1. 打开工具网站，输入你的 VPS IP 地址和 SSH 端口号（如果你不知道端口号，可以在 KiwiVM 面板首页查看）。
2. 等待扫描结果，对照下面的判断逻辑。

**结果判断逻辑：**

| 国外结果 | 国内结果 | 结论 |
|---------|---------|------|
| 可达 | 可达 | IP 完全正常，检查本地网络 |
| 可达 | 不可达 | IP 被墙，需要换 IP 或换机房 |
| 不可达 | 不可达 | VPS 本身问题（未开机、系统挂掉等），与 IP 无关 |

有一点老用户都知道：**只看 ping 已经不够用了**。因为有些情况下 IP 可以 ping 通，但 TCP 端口（也就是 SSH 连接端口）被单独封了。所以正确的判断方式是：**既要看 ICMP（ping），也要看 TCP 端口**。如果国内 ping 通但 SSH 连不上，很可能是端口被封，先试试换个 SSH 端口，不行再考虑换 IP。

## 方法三：检测结果解读矩阵——对号入座，省时省力

把前两种方法结合起来，检测结果无非就是这几种组合：

**情况 A：KiwiVM 显示 IP NOT BLOCKED，境外测试可达，国内测试不可达**
→ IP 被墙了。先尝试换端口，如果换端口无效就考虑付费换 IP 或迁移机房。

**情况 B：KiwiVM 显示 IP BLOCKED**
→ 被搬瓦工官方封禁，必须付费换 IP（$8.79），没有其他选项。

**情况 C：境外测试不可达，国内也不可达**
→ VPS 本身出了问题，和 IP 没关系。先去 KiwiVM 面板检查 VPS 是否正在运行，尝试重启，必要时重装系统。

**情况 D：境外和国内都可达，但 TCP 端口不通**
→ 端口被封，而不是 IP 被封。到 KiwiVM 面板修改 SSH 端口，换一个端口测试。

**情况 E：什么都正常，但还是连不上**
→ 考虑本地网络问题。试试手机热点连接，或者换个时间段再试。国内高峰期（晚上 8-11 点）对线路稳定性影响较大，有时检测结果也不准，建议不同时间段测两三次。
